## 一、真实 NAS 回归清单（必须）

### A. 基础连通与预检

1. 打开“添加 WebDAV 源”，输入真实 NAS 地址（http/https）。
2. 点击“测试连接”：
   - 正常账号：应显示连接成功（含 HTTP 状态）。
   - 错误密码：应显示认证失败（401/403 类）。
3. 失败后关闭弹窗，首页工具栏应出现“复制诊断”按钮；点击后应可粘贴出结构化文本。

验收标准：
- 成功/失败提示与实际情况一致。
- 诊断文本不包含明文密码（URL 已脱敏）。

### B. 扫描与刮削

1. 保存 WebDAV 源后执行“同步”。
2. 观察进度文本应显示按源扫描状态。
3. 验证媒体文件入库、刮削、海报与剧照链路。
4. 验证包含中文/空格/特殊符号（`:`, `?`, `*` 等）路径的文件是否能被识别。

验收标准：
- 无异常崩溃。
- 文件入库数量符合预期，未出现误删历史记录。

### C. 播放与续播点

1. 选择 WebDAV 媒体播放，执行拖动、暂停、退出、恢复播放。
2. 验证续播点写入与恢复是否正常。
3. 验证剧集切换（下一集）行为。

验收标准：
- 可稳定播放、seek、恢复播放。
- 无明显卡死/黑屏/音画错位。

### D. 异常场景

1. 扫描中断网（或禁用 NAS WebDAV 服务）后重试。
2. 模拟服务端 5xx（如果 NAS/反代可配置）。
3. 错误账号重复尝试后恢复正确账号。

验收标准：
- 错误分类合理（network/server/auth）。
- 复制诊断可用于排障（包含源名、分类、HTTP/URLError、端点、消息）。
